# 05 AI 工作流设计

## 一、不要一个 Agent 干所有事

推荐拆分为多个稳定步骤：

```text
需求解析 Agent
教学设计 Agent
PPT 大纲 Agent
逐页内容 Agent
语言审查 Agent
教学审查 Agent
PPT 渲染器
```

---

## 二、LLM 调用策略

### mock

用于开发，不依赖模型。

### ollama

用于本地生成草稿，成本低，适合日常使用。

### deepseek / openai

用于精品课件、公开课、最终润色。

---

## 三、提示词要求

必须要求模型输出 JSON。

示例：

```text
You are a senior high school English teacher.
Generate a classroom-ready PPT structure.
Return valid JSON only.
Do not include markdown.
```

---

## 四、质量审查规则

生成后要检查：

1. 是否符合课型；
2. 是否有教学目标；
3. 是否有课堂互动；
4. 每页内容是否过多；
5. 英文是否适合高中生；
6. 授课备注是否可执行；
7. 是否有总结和作业；
8. 是否有板书设计。

---

## 五、重新生成单页

不要重新生成整份课件。流程：

```text
读取当前 task 信息
读取当前 slide
读取前后 slide 上下文
调用 AI 重新生成当前页
保存当前页
```

这样效率高，也不会破坏整份课件结构。
